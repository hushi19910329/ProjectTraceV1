# 项目管理平台页面设计文档

## 1. 文档说明

- 文档名称：项目管理平台页面设计文档
- 输出位置：`PageDesign/`
- 设计目标：将需求文档中的功能进一步拆解为可落地的页面结构，先形成低保真静态页面草图，后续再进入 Vue3 组件化开发
- 设计风格：后台管理系统风格，使用 Element 系浅色基础布局，不追求视觉精修，重点表达页面结构、信息层级、关键操作和模块关系

## 2. 页面设计原则

- 以信息透明为核心，尽量减少隐藏入口
- 首页突出待办、风险、目标、项目动态
- 项目详情页承担“过程晾晒”作用
- 需求、任务、缺陷、版本都强调状态流转与关联关系
- 每个核心对象都支持“列表页 + 详情页 + 关联信息区”
- 尽量保留顶部快捷操作区、筛选区、主数据区、侧边信息区

## 3. 页面结构总览

建议系统采用以下导航结构：

1. 登录页
2. 工作台
3. 项目中心
4. 项目详情
5. 需求中心
6. 需求详情
7. 任务详情页
8. Sprint / 看板页
9. 任务日历 / 时间线
10. 测试与缺陷中心
11. 工时与日报中心
12. OKR 与透明化中心
13. 版本与发布中心
14. 统计报表中心
15. 消息与待办中心
16. 文档与知识库
17. 组织与系统管理

## 4. 页面清单与说明

### 4.1 `01-index.html`

- 用途：页面草图总入口
- 主要内容：项目说明、页面目录、设计约定、跳转链接

### 4.2 `02-login.html`

- 用途：账号登录
- 目标角色：全体用户
- 关键区域：登录表单、平台简介、最近公告

### 4.3 `03-dashboard.html`

- 用途：个人工作台 / 平台首页
- 目标角色：全体用户
- 关键区域：快捷创建、今日待办、我的项目、风险提醒、项目动态、OKR 概览

### 4.4 `04-project-list.html`

- 用途：项目中心列表页
- 关键区域：筛选区、项目统计卡片、项目表格、批量操作区

### 4.5 `05-project-detail.html`

- 用途：项目详情与过程晾晒页
- 关键区域：项目头部摘要、目标与里程碑、项目动态时间线、风险问题阻塞、成员资源

### 4.6 `06-requirement-center.html`

- 用途：需求中心 / Backlog
- 关键区域：待办池、筛选区、需求列表、评审安排区

### 4.7 `07-requirement-detail.html`

- 用途：需求详情页
- 关键区域：基本信息、业务价值、评审结论、关联任务测试缺陷版本、变更记录、评论动态

### 4.8 `08-sprint-board.html`

- 用途：Sprint 管理与任务看板
- 关键区域：Sprint 头部信息、燃尽图占位、每日站会摘要、看板列、WIP 提示、侧边任务详情

### 4.9 `09-task-detail.html`

- 用途：任务详情页
- 关键区域：任务基本信息、任务树、技术线子任务、执行人、附件素材、评论动态、工时记录、操作记录

### 4.10 `10-task-calendar.html`

- 用途：任务日历 / 时间线
- 关键区域：周月切换、成员排期区、甘特视图占位、延期任务区

### 4.11 `11-test-bug-center.html`

- 用途：测试与缺陷中心
- 关键区域：测试计划、测试用例列表、缺陷趋势摘要、缺陷列表、回归状态区

### 4.12 `12-timesheet-center.html`

- 用途：工时、日报、资源负载页
- 关键区域：工时填报区、我的日报、团队工时统计、资源负载看板

### 4.13 `13-okr-center.html`

- 用途：OKR 与目标透明化页
- 关键区域：目标地图、O/KR 列表、目标关联项目、风险与阻塞晾晒、达成趋势

### 4.14 `14-release-center.html`

- 用途：版本与发布中心
- 关键区域：版本列表、发布检查项、发布窗口、发布记录、上线验证

### 4.15 `15-report-center.html`

- 用途：统计报表中心
- 关键区域：统计筛选区、项目健康度、需求趋势、任务趋势、缺陷趋势、工时投入、透明度分析

### 4.16 `16-message-center.html`

- 用途：消息、待办、通知中心
- 关键区域：待我处理、抄送我的、项目动态订阅、评论@消息、里程碑提醒

### 4.17 `17-doc-center.html`

- 用途：文档与知识库中心
- 关键区域：文档目录树、文档列表、文档预览、关联对象、知识沉淀入口

### 4.18 `18-admin-center.html`

- 用途：组织与系统管理页
- 关键区域：用户管理、部门管理、角色模板、自定义字段、状态流、操作记录

## 5. 页面关系建议

- 工作台进入各业务中心
- 项目中心进入项目详情
- 项目详情联动需求、Sprint、缺陷、版本、文档
- 需求中心进入需求详情
- 需求详情进入任务详情
- 任务详情承载任务树、子任务、技术线子任务、附件和执行记录
- Sprint 看板与任务详情、任务日历共享任务数据
- 报表中心汇总项目、需求、任务、缺陷、工时、OKR 数据

## 6. HTML 草图说明

- 当前 HTML 只表达布局和模块层级
- 当前采用基础块状结构和表格表达
- 后续 Vue3 实现时建议拆分成布局、筛选、列表、详情、看板卡片、统计卡片、时间线等组件
