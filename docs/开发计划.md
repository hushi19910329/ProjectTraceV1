# 开发计划

## 1. 开发目标

本项目目标是在保证业务清晰、架构可扩展、部署可落地的前提下，逐步完成一个可用于公司内部的项目管理平台 MVP，并在后续持续扩展。

## 2. 开发总体策略

建议采用以下策略：

- 先文档后实现
- 先骨架后功能
- 先 MVP 闭环后增强
- 先本地容器化联调后部署上线
- 每完成一个阶段，就同步更新文档、TODO、开发记录

## 3. 开发阶段拆分

## 阶段 1：规划与设计阶段

目标：

- 明确需求
- 明确流程
- 明确页面结构
- 明确系统架构

交付物：

- 需求文档
- 业务流程文档
- 页面设计文档
- 系统架构文档
- 开发计划文档
- TODO 文档

当前状态：

- 进行中，且大部分已完成

## 阶段 2：技术设计阶段

目标：

- 确定数据库模型
- 确定后端模块划分
- 确定前端路由结构
- 确定 Docker Compose 运行方案

具体任务：

1. 设计 PostgreSQL 表结构
2. 设计文件型数据库结构
3. 设计 Redis 使用场景
4. 设计 FastAPI 模块目录
5. 设计 Vue3 项目目录和路由
6. 设计接口规范和统一响应格式

交付物：

- 数据库设计文档
- API 设计文档
- 前端路由与组件规划文档
- 容器部署草案

## 阶段 3：项目骨架初始化阶段

目标：

- 创建前后端基础工程
- 建立统一开发环境
- 建立容器化部署环境

具体任务：

1. 初始化后端 FastAPI 项目
2. 初始化前端 Vue3 项目
3. 建立基础目录结构
4. 接入 PostgreSQL
5. 接入文件型数据库
6. 接入 Redis
7. 编写 `docker-compose.yml`
8. 建立环境变量文件
9. 配置本地开发说明

交付物：

- 可运行的前后端项目骨架
- 可启动的 Docker Compose 环境

## 阶段 4：基础能力开发阶段

目标：

- 先打通系统基础能力

优先开发模块：

1. 登录认证
2. 用户、部门、角色、项目成员基础管理
3. 项目管理
4. 文档上传与附件管理基础能力
5. 基础消息通知能力

交付物：

- 可以登录
- 可以创建项目
- 可以配置项目成员
- 可以上传并查看基础附件

## 阶段 5：项目执行闭环开发阶段

目标：

- 打通核心项目管理闭环

优先开发模块：

1. 需求管理
2. 主任务管理
3. 子任务管理
4. 技术线子任务管理
5. 执行人分配与状态流转
6. 任务评论与操作记录
7. 工时记录

交付目标：

- 项目 -> 需求 -> 主任务 -> 子任务 -> 技术线子任务全链路可用

## 阶段 6：测试、缺陷、发布闭环阶段

目标：

- 补齐研发交付全过程

优先开发模块：

1. 测试计划
2. 测试用例
3. 缺陷管理
4. 版本管理
5. 发布记录
6. 上线验证

交付目标：

- 从开发执行到测试缺陷，再到版本发布形成闭环

## 阶段 7：透明化与管理提升阶段

目标：

- 强化项目过程晾晒、报表统计与 OKR 对齐

优先开发模块：

1. 项目动态时间线
2. 看板与时间线
3. 统计报表
4. OKR 管理
5. 风险与问题晾晒
6. 资源负载分析

交付目标：

- 平台从“任务工具”升级为“透明协作平台”

## 阶段 8：优化与上线阶段

目标：

- 提升稳定性、性能、可维护性

具体任务：

1. 权限与范围控制优化
2. 日志与监控补充
3. 接口性能优化
4. 前端体验优化
5. 部署脚本与上线文档完善
6. 灰度和正式环境部署

## 4. MVP 建议顺序

建议优先完成以下 MVP 路线：

1. 登录与用户体系
2. 项目管理
3. 需求管理
4. 主任务 / 子任务 / 技术线子任务
5. 附件与素材管理
6. 工时与动态记录
7. 测试与缺陷
8. 版本与发布
9. 基础报表

## 5. 开发模式建议

- 后端先定义数据模型和接口契约
- 前端根据页面草图与接口契约开发
- 先做假数据联调，再接真实接口
- 每个模块完成后进行一次文档回写

## 6. Docker Compose 开发推进建议

建议最早在“项目骨架初始化阶段”就引入 Docker Compose，而不是等开发后期再补。

推荐容器：

- `frontend`
- `backend`
- `redis`

后续可扩展：

- `nginx`
- `celery_worker`
- `celery_beat`

## 7. 里程碑建议

### 里程碑 1：完成设计文档

- 标志：需求、流程、页面、架构、计划文档齐全

### 里程碑 2：完成项目骨架

- 标志：前后端工程可启动，PostgreSQL、文件型数据库和可选 Redis 可连通

### 里程碑 3：完成核心任务闭环

- 标志：项目、需求、任务、子任务、技术线子任务流程可用

### 里程碑 4：完成测试发布闭环

- 标志：缺陷、版本、发布功能可用

### 里程碑 5：完成透明化与报表能力

- 标志：看板、时间线、统计、OKR 可用
